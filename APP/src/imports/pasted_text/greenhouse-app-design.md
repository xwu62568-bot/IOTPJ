2. 核心信息架构
2.1 核心对象
项目
* 项目是业务管理的核心单位。
* 一个项目隶属于一个分区。
* 项目下可关联：
    * 设备

分区
* 分区用于对项目进行归类管理。

成员与权限
* 每个项目下有成员管理。
* 不同成员拥有不同权限。

5.4 实时页卡片结构
实时页采用卡片网格形式展示关键数据，卡片既可展示实时值，也可作为详情/配置入口。
卡片类型
1. 温度卡片
2. 湿度卡片
3. 二氧化碳卡片
4. VPD 卡片
5. 灯光卡片
5.5 实时卡片详细定义
A. 温度卡片
用途： 展示当前空气温度。
体现：
* 当前值，如 16.9℃
* 告警角标（L）
* 红色背景表示异常或偏离目标区间
交互：
* 点击进入“温度设置”页
* 可查看日/夜间制冷加热阈值、联动设置、回差
B. 湿度卡片
用途： 展示当前空气湿度。
体现：
* 当前值，如 62.0%
* 蓝色数字强调数值
交互：
* 点击进入湿度相关详情或联动配置页
C. 二氧化碳卡片
用途： 展示当前 CO₂ 浓度。
体现：
* 当前值，如 600 PPM
* 橙黄色数字强调数值
交互：
* 点击进入“二氧化碳设置”页
* 可配置白天补充、夜间减少、修正值、回差、联动逻辑
D. VPD 卡片
用途： 展示当前蒸汽压亏缺（VPD）值。
体现：
* 当前值，如 0.73 kPa / 0.61 kPa
* 红色背景并带告警角标
* 点击后进入 VPD 详情页
交互：
* 查看当前温度与湿度对 VPD 的计算结果
* 展示推荐区间
* 支持图表入口
E. 灯光卡片
用途： 展示灯光状态或光照数据。
体现：
* 卡片标题为“灯光”
* 单位为 PPFD
* 无数据时显示 -- PPFD
交互：
* 点击进入灯光控制配置页
* 支持调光/恒光、定时/循环、温度联动、模拟日升日落

5.6 告警状态表达
定义：
* L：低于阈值 / 低告警
* H：高于阈值 / 高告警
* 灰色：离线/无数据
* 蓝色：正常
* 红色底：异常
后续可形成统一告警视觉规范。
5.7 实时卡片点击后的配置页

* 二氧化碳设置页

* 灯光控制配置页
5.8 温度设置页
页面目标： 管理温度阈值与温控联动。
* 白天卡片
    * 制冷温度
    * 加热温度
* 夜晚卡片
    * 制冷温度
    * 加热温度
* 制冷和除湿关联开关
* 温度回差
* 保存
规则含义整理：
* 白天和夜晚采用不同温度控制阈值
* 可开启“制冷和除湿关联”
* “温度回差”用于避免设备频繁启停

5.9 二氧化碳设置页
页面目标： 管理 CO₂ 补充与联动规则。
结构：
* 白天目标
    * 补充二氧化碳目标值（PPM）
    * 模糊控制开关（适用减压阀）
    * 二氧化碳和制冷关联开关
    * 二氧化碳和除湿关联开关
* 夜晚目标
    * 减少二氧化碳目标值（PPM）
* 二氧化碳手动修正
* 二氧化碳回差
* 保存
规则含义整理：
* 白天与夜晚 CO₂ 控制策略不同
* CO₂ 与制冷/除湿存在联动关系
* 支持修正值补偿
* 支持回差控制避免频繁开关
5.10 灯光控制配置页
页面目标： 对灯光通道进行计划控制和亮度控制。
结构：
* 通道切换（通道1 / 通道2）
* 使用的灯光类型（如下拉选择 LED）
* 计划模式切换：
    * 定时计划
    * 循环
* 首次启动时间
* 开启时长
* 关闭时长
* 灯光模式切换：
    * 调光
    * 恒光
* 调光模式
    * 滑杆百分比控制（如 80%）
* 恒光模式
    * 目标 PPFD 值（如 1200 PPFD）
* 调光温度阈值（温度超过）
* 关闭温度阈值（温度超过）
* 模拟日升日落（分钟）
* 保存
规则含义整理：
* 一个灯控设备可能有多个通道
* 每个通道可以独立配置
* 灯光支持两种控制方式：
    * 调光：直接设定亮度百分比
    * 恒光：以目标 PPFD 自动控制
* 灯光与温度存在保护联动
* 支持模拟日出日落，适合植物照明场景
5.11 VPD 详情页
页面目标： 展示 VPD 当前值、推荐范围及相关环境参数。
结构：
* 中央大仪表/圆环显示当前 VPD
* VPD 图表 按钮
* 下方分区说明：
    * 生长期 VPD：0.80 ~ 0.95
    * 开花期 VPD：0.96 ~ 1.15
    * 应激 VPD：1.16 ~ 1.35
    * 亚最适 VPD
* 底部关联参数：
    * 温度
    * 湿度
规则含义整理：
* VPD 页面更偏“分析页”而非纯配置页
* 需要根据温湿度计算 VPD
* 需要为种植阶段提供推荐区间参考
5.12 实时页水下的项目新增：复合卡片（多传感器聚合）
实时页中新增一种非常关键的卡片类型：复合卡片（容器/系统级卡片）。
5.12.1 复合卡片定义
该类卡片不是单一指标，而是一个业务对象（如水箱/营养池）下的多个传感器数据集合展示。
5.12.2 卡片结构
卡片内部按区域分组：
A. 营养池（主水体）
* 营养池 pH
* 营养池 EC（mS/cm）
* 营养池温度（℃）
B. 管道
* 管道 pH
* 管道 EC
* 管道温度
（无数据时显示 --）
C. 水位与基质
* 水位（m）
* 基质湿度（%）
* 基质温度（℃）
5.12.3 特点总结
* 一个卡片 = 一个“控制对象”（如区域、系统）
* 一个对象下绑定多个传感器
* 按物理/业务分区展示（营养池 / 管道 / 基质）
* 支持部分数据缺失（--）
5.12.4 卡片操作
点击卡片 → 进入“系统配置页”（见下）

5.13.6 安全保护策略
* EC 高保护（阈值）
* pH 低保护

超过阈值时触发保护（停止加肥/补水等）

* ON：只监控，不参与自动控制
* OFF：参与自动控制逻辑
这是一个非常关键的控制权限开关。

目标： 查看项目下所有传感器的数据来源、实时状态、指标内容与异常情况。
7.1 页面结构

    * 平均值

7.2 传感器页业务含义
说明“实时首页”的某些环境卡片数据，并不一定固定来自单一传感器，而是支持两种来源方式：
1. 平均值：多个同类传感器取平均后作为首页展示值
2. 指定传感器：指定某个具体传感器作为首页展示来源
这对专业温室项目非常重要，尤其适用于一个项目内存在多个温湿度/CO₂/光照探头的情况。
7.3 传感器卡片列表结构
每个传感器卡片当前可识别字段包括：
* 传感器 ID（如 ID:25 / ID:24 / ID:29）

* 实时数据指标
* 图表按钮
* 定位/配置按钮（部分卡片）
* 静音/屏蔽按钮（部分卡片）
* 卡片状态颜色
7.4 传感器类型识别
当前至少支持以下传感器类型：
A. 温湿度 + CO₂ 复合传感器
卡片展示：
* CO₂（PPM）
* 湿度（%）
* 温度（℃）
* 第四项扩展值（数值 51 / 585，需后续确认字段）
这个第四项可能是以下其中一种：
* 光照
* TVOC
* 海拔/气压
* 设备信号强度
* 传感器附加通道值
当前先在需求中标记为：扩展参数位，等待后续确认。
B. PAR / 光量子传感器
卡片展示：
* PPFD
示例：
* 0 PPFD
说明系统已支持针对植物补光场景的光照采集。
7.5 传感器卡片状态表达
不同卡片底色：
* 灰色：可能表示离线、被禁用、无效数据、未参与首页统计
* 蓝色：正常在线
建议规则：
* 蓝色：在线正常
* 灰色：离线/停用/未启用
* 红色边角或角标：存在告警
* 黄色：弱信号/数据异常
